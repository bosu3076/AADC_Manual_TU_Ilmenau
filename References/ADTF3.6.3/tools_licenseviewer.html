License Viewer

The licenseviewer.exe is able to request a license for ADTF 3 via email, load or borrow from a network server. It also shows the available license features. The licenseviewer.exe has the following options:

  • Request License: If a mailing programm is installed, the application creates an email containing your MAC addresses.
  • Load License: Select and load your ADTF 3 license.
  • Server License: If you are using an ADTF license server, enter your details and click the "save" button.
  • Borrow License: If you want to use a borrow license, enter your details and click the "save" button.
  • Show license features: Shows the available license features.

    • .lic files for ADTF 2 can only be used within ADTF 2.x, .lic files for ADTF 3 can only be used with ADTF 3.x. There is no compatibility between these two major versions, like the upgrade from ADTF 1.x to ADTF 2.x.

    Getting Licensed

    To get a licence follow these steps:

    1. Start the program licenseviewer.exe which you can find inside the bin folder of the ADTF directory
    2. Make a license request by clicking on Request License
    3. Either an email is created including most of the required information for the request or you can paste it into your mail program hitting ctrl + V. Just complete the information about:
      • Organization & Department
      • Responsible person e.g. the project manager
      • Additional information if any (e. g. the project name, version)
    4. Send the email
    5. Wait until you receive the license
    6. Start the program licenseviewer.exe and choose Load License select the license file that was attched to the email you received

    Command Line usage

    The licenseviewer.exe provides command line support as well, e.g. to load a .lic file in a script before starting an adtf instance. The following lists all options the tool supports. Just issue the following command: licenseviewer.exe --help 

                    
                    
    
                        
    usage:
    
                        
      licenseviewer.exe  options
    
                        
     
    
                        
    where options are:
    
                        
    -?, -h, --helpDisplay usage information
    
                        
    -t, --type <type>License type [borrow, server, file]
    
                        
    -s, --server <server>License server url
    
                        
    -p, --port <port>License server port
    
                        
    -v, --volume <volume>Covered feature scope [nofeature, runtime, developer]
    
                        
    -d, --days <borrow>Days of borrow
    
                        
    -f, --file <file>license file to load

    Loading license within your application

    If your application depends on licensed plugins and you are not using the ADTF-Launcher, you must set an environment variable. The internal license mechanism is not able to load the corresponding library. ADTF offers for this approach the environment variable ADTF_LICENSER_PLUGIN. This environment variable has to be set to the corresponding license file.

    ADTF_LICENSER_PLUGIN = "$(ADTF_DIR)/bin/licenser.dll" The environment variable must not point to you *.lic-file, which contains your specific library. The environment variable points to the plugin (should be in the bin-directory)
