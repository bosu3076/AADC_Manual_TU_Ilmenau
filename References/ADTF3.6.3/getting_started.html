Getting Started

This guide covers getting up and running with ADTF. After reading this guide, you will:

  • know where the delivered tools are located
  • know the different type of meta files used within ADTF System
  • know how to launch a simple ADTF Session
  • have a basic understanding what ADTF can do for you

What is ADTF?

ADTF is an application development framework written in the C++ language. It is designed to record data generated from all kind of data sources you find in modern cars, transform, combine or manipulate this data and supply it to any kind of data sink. It allows you to include new devices with little coding effort while making use of a rich set of default components like video or image display, recoder and player.

What are the building blocks of ADTF?

ADTF System

An ADTF System encapsulates all things necessary to get an ADTF Session up and running on different environments.

  • ADTF Plugins (.adtfplugin)
  • ADTF Session Services (.adtfplugin)
  • ADTF System Services (.adtfplugin)
  • Platform specific dependencies (shared objects or dynamic libraries used by .adtfplugins)
  • Environment variables

All of this is saved within a ADTF System File which can be recognized by the file ending .adtfsystem.

ADTF Streaming Graph

The ADTF Streaming Graph defines the ADTF System's communication interface to

  • other ADTF instances
  • non ADTF Applications
  • devices like cameras, CAN devices, ethernet devices,...
  • recording and playback of ADTF DAT Files

The definition is made within a ADTF Graph File (.adtfgraph) and can contain one or more ADTF Streaming Graphs.

ADTF Filter Graph

The ADTF Filter Graph defines how data and trigger flow through a graph based system where you can

  • manipulate data
  • transforming data
  • calculate data
  • displaying data

The definition is made within a ADTF Graph File (.adtfgraph) and can contain one or more ADTF Filter Graphs.

ADTF Session

The ADTF Session is the file you need in order to launch an ADTF Launcher. Therefore the session selects:

  • the ADTF System to use (what to load)
  • the active ADTF Streaming Graph (where to get and put streaming data)
  • the active ADTF Filter Graph (what to do)

The definition is made within a ADTF Session File (.adtfsession).

ADTF Project

The ADTF Project contains all kind of files described previously to run an ADTF Launcher. An ADTF Project may contain multiple ADTF Filter Graphs ,ADTF Sessions, ADTF Systems but at least one:

  • ADTF System File (.adtfsystem)
  • ADTF Graph File (.adtfgraph)
  • ADTF Session File (.adtfsession)
in order to an ADTF Launcher.

The definition is made within a ADTF Project File (.adtfproject).

How can I launch it?

Let's launch a little example session:

  1. Open a console and go into the bin directory and start the Configuration Editor by typing: adtf_configuration_editor.exe
  2. Next open the example project by pressing ctrl + O
  3. Enter the path to the project file ..\src\examples\projects\ADTF_Project\ADTF Project.adtfproject and press open
  4. Right click on Playback Example and choose Launch via ADTF Control

  5. Type rl running into the console

What do I see?

The ADTF Runtime is now in the state running. The loaded ADTF Session replays a previously recorded data stream. The ADTF XSystem, a mashup and handler of different widgets, displays various data contained in the stream and logging. Furthermore a player control enables the user to start, stop or seek the stream.

The XSystem is a modular view, maybe it won't appear exactly as you can see in the following picture. This means some widgets are maybe missing or the layout is different. Feel free to move widgets by holding the task bar of it to drag an drop it to different spaces, tab it to other widgets or whatever you like. If some widgets are not visible there are two possibilities:
  • The defined runlevel is not already reached, so please increase the runlevel
  • The widget visibility is disabled, so please right-click on any free space in the XSystem Main Window and enable/disable the widgets you like or dislike
